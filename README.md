# Real-Time Data Pipeline (Kafka + Spark)

## 📌 Overview
Designed a real-time data processing system using Kafka and Spark Structured Streaming.

## ⚙️ Architecture
Producer → Kafka → Spark Streaming → PostgreSQL

## 📊 Data Flow
Data is produced by source systems → sent to Kafka topics → processed in real-time via Spark Structured Streaming → stored in PostgreSQL for downstream analytics and reporting.

## 🚀 Features
- Real-time data ingestion
- Scalable stream processing
- Fault-tolerant pipeline design
- Structured Streaming processing layer
- Database storage for analytics and reporting

## 🛠 Technologies
- Apache Kafka
- Apache Spark / PySpark
- Python
- PostgreSQL
- Linux

## 📊 Business Value
This project demonstrates how real-time data can be collected, processed, and stored efficiently for analytics, reporting, and operational decision-making.
